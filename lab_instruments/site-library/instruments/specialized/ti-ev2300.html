    TI EV2300 (USB-to-I2C)¶

    USB-to-I2C/SMBus adapter for battery fuel gauge communication. Interface: USB HID.

    from lab_instruments import TI_EV2300

    Optional Dependency

    Requires hidapi on Linux/macOS: pip install hidapi. On Windows, uses native HID APIs via ctypes. If unavailable, TI_EV2300 will be None when imported.

    Not SCPI-Based

    This driver communicates via USB HID protocol, not PyVISA/SCPI. It does not inherit from DeviceManager.

    Quick Example¶

    from lab_instruments import TI_EV2300

ev = TI_EV2300()
ev.connect()

try:
    # Read a 16-bit word from I2C address 0x16, register 0x09
    result = ev.read_word(0x16, 0x09)
    if result["ok"]:
        print(f"Voltage: {result['value']} mV")
    else:
        print(f"Error: {result['error']}")
finally:
    ev.disconnect()

    Methods¶

    Connection¶

    Method Description
    connect() Open the HID device
    disconnect() Close the HID device
    is_open() Check if device is connected

    SMBus Read Operations¶

    Method Returns Description
    read_word(i2c_addr, register) dict Read 16-bit word. Returns {"ok": bool, "value": int, "error": str}
    read_byte(i2c_addr, register) dict Read 8-bit byte
    read_block(i2c_addr, register) dict Read block of bytes. Returns {"ok": bool, "data": bytes, "length": int}

    SMBus Write Operations¶

    Method Returns Description
    write_word(i2c_addr, register, value) dict Write 16-bit word
    write_byte(i2c_addr, register, value) dict Write 8-bit byte
    write_block(i2c_addr, register, data) dict Write block of bytes
    send_byte(i2c_addr, command) dict Send single byte command

    Diagnostics¶

    Method Returns Description
    get_device_info() dict Device information (VID, PID, serial, etc.)
    probe_command(i2c_addr, start_reg, end_reg, ...) list[dict] Scan register range

    Static Methods¶

    Method Returns Description
    TI_EV2300.enumerate_devices() list[dict] List all connected EV2300 devices
    TI_EV2300.is_available() bool Check if any EV2300 is connected

    Return Value Format¶

    All read/write operations return a dict with:

    {
    "ok": True,       # Whether the operation succeeded
    "value": 1234,    # For read_word/read_byte: the integer value
    "data": b"\x01",  # For read_block: the raw bytes
    "length": 2,      # For read_block: number of bytes
    "error": "",      # Error message if ok=False
}

    Register Scan Example¶

    ev = TI_EV2300()
ev.connect()

try:
    # Scan registers 0x00-0x1F on address 0x16 (standard SBS fuel gauge)
    results = ev.probe_command(0x16, 0x00, 0x1F)
    for r in results:
        if r["ok"]:
            print(f"Reg 0x{r['register']:02X}: {r['value']}")
finally:
    ev.disconnect()

    Battery Fuel Gauge Example¶

    ev = TI_EV2300()
ev.connect()

try:
    GAUGE_ADDR = 0x16  # Standard SBS address

    # Read standard SBS registers
    voltage = ev.read_word(GAUGE_ADDR, 0x09)    # Voltage (mV)
    current = ev.read_word(GAUGE_ADDR, 0x0A)    # Current (mA, signed)
    soc = ev.read_word(GAUGE_ADDR, 0x0D)        # State of Charge (%)
    temp = ev.read_word(GAUGE_ADDR, 0x08)        # Temperature (0.1K)

    if voltage["ok"]:
        print(f"Voltage: {voltage['value']} mV")
    if current["ok"]:
        # Current is signed 16-bit
        raw = current["value"]
        signed = raw - 65536 if raw > 32767 else raw
        print(f"Current: {signed} mA")
    if soc["ok"]:
        print(f"SOC: {soc['value']}%")
    if temp["ok"]:
        temp_c = temp["value"] / 10.0 - 273.15
        print(f"Temperature: {temp_c:.1f} C")
finally:
    ev.disconnect()

    Notes¶

    • Cross-platform: Windows (native ctypes), Linux (/dev/hidrawN), macOS (hidapi or IOKit).
    • USB VID: 0x0451 (Texas Instruments), PID: 0x0036 (operational mode).
    • The EV2300 does not require any special drivers on Windows — it uses the built-in HID driver.
    • PEC (Packet Error Checking) is supported on some operations via the pec parameter.
